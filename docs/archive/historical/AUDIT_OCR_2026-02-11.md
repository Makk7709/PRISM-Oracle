# Audit OCR — KOREV Evidence

**Date :** 2026-02-11  
**Constat :** L'OCR ne lit aucun document, même de bonne qualité.  
**Objectif :** Identifier et corriger les causes.

---

## 1. Architecture OCR

| Composant | Fichier | Rôle |
|-----------|---------|------|
| OCREngine | `python/helpers/pdf_extraction/ocr_engine.py` | Moteur central : Tesseract + pdf2image |
| Tool PDF OCR | `python/tools/pdf_ocr.py` | Outil agent : extraction directe puis OCR si ≤ 50 car. |
| Document Query | `python/helpers/document_query.py` | PyPDF puis fallback OCR si vide |
| Pipeline | `python/helpers/pdf_extraction/pipeline.py` | OCR si config.enabled + type "scan" + word_count < 10 |

---

## 2. Causes identifiées

### 2.1 Poppler manquant dans Docker (CRITIQUE)

**Problème :** `pdf2image.convert_from_path()` nécessite **poppler-utils** (pdftoppm). Le `deploy/Dockerfile.backend` n'installe que tesseract, pas poppler.

**Symptôme :** En production Docker, `convert_from_path` lève une exception (pdftoppm not found).

**Correction :** Ajouter `poppler-utils` dans le RUN apt-get du Dockerfile.

---

### 2.2 Condition OCR trop stricte dans document_query

**Problème :** `if not contents:` — si PyPDF retourne une chaîne de whitespace (`"\n\n  \n"`), la condition est False et l'OCR n'est jamais appelé.

**Correction :** `if not contents.strip():`

---

### 2.3 Seuil OCR dans pdf_ocr tool

**Problème :** `if len(text.strip()) > 100` — si le PDF a 51–100 caractères extraits (métadonnées, texte partiel), on considère qu'il n'est pas scanné.

**Correction :** Seuil réduit à 50 pour lancer l'OCR plus souvent en cas de doute.

---

### 2.4 Pipeline OCR désactivé par défaut

**Problème :** `config.ocr.enabled = False` et `only_if_pdf_type = ["scan"]`. Le pipeline ne lance l'OCR que si le PDF est classifié "scan" et que l'extraction texte donne < 10 mots.

**Impact :** Le pipeline n'est pas le point d'entrée principal pour document_query — le fallback OCR dans document_query est indépendant. Pas de correction immédiate nécessaire.

---

## 3. Corrections appliquées

| # | Correction | Fichier |
|---|------------|---------|
| 1 | Ajouter `poppler-utils` dans le runtime stage | `deploy/Dockerfile.backend` |
| 2 | `if not contents.strip()` (whitespace fallback OCR) | `python/helpers/document_query.py` |
| 3 | Seuil 100 → 50 caractères pour déclenchement OCR | `python/tools/pdf_ocr.py` |
| 4 | `libgdk-pixbuf2.0-0` → `libgdk-pixbuf-2.0-0` (Debian Trixie) | `deploy/Dockerfile.backend` |
| 5 | `openai-whisper` : `20240930` → `20250625` (fix `pkg_resources`) | `requirements.txt` |
